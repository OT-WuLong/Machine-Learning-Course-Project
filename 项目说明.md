# 机器学习课程设计 —— 学生成绩预测

---

## 一、项目概述

使用某课程学生成绩数据，训练两种回归模型，实现对学生总成绩的预测。

| 项目 | 内容 |
|:----|:----|
| 任务类型 | 模型 A：回归（预测学生总成绩）<br>模型 B：分类（预测成绩等级：优/良/中/差） |
| 模型 A | 多元线性回归（Multiple Linear Regression） |
| 模型 B | 决策树分类（Decision Tree Classifier） |
| 数据量 | 512 条学生记录，来自 7 个班级 |
| 交互方式 | 顶部导航栏 + 勾选框选特征 + 滑块/数字输入 |

## 三、算法层

### 3.1 模型 A：多元线性回归

#### 算法原理

多元线性回归试图用多个自变量（特征）的线性组合来预测因变量（总成绩）：

$$
y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_n x_n + \varepsilon
$$

其中 $\beta_0$ 为截距，$\beta_i$ 为各特征的回归系数（权重），$\varepsilon$ 为误差项。训练过程通过 **最小二乘法（OLS）** 最小化预测值与真实值之间的残差平方和。

#### 训练过程

```python
from sklearn.linear_model import LinearRegression

model_a = LinearRegression()
model_a.fit(X_train, y_train)
```

模型无须额外超参数，直接拟合训练数据。

#### 回归系数分析

| 特征 | 权重 (Weight) | 相关性方向 |
|:----|:------------:|:---------:|
| 线上期末考 | **1.1260** | 正相关 |
| 线上章测试 | **0.7401** | 正相关 |
| 线上平时成绩 | **0.6039** | 正相关 |
| 班级_Sheet5 | **-0.5810** | 负相关 |
| 线下期末考 | **0.3201** | 正相关 |
| 班级_Sheet6 | -0.1429 | 负相关 |
| 班级_Sheet3 | -0.1042 | 负相关 |
| 线下互动 | 0.0563 | 正相关 |
| 班级_Sheet7 | 0.1079 | 正相关 |
| 班级_Sheet2 | -0.0240 | 负相关 |
| 班级_Sheet4 | -0.0079 | 负相关 |

回归方程：

```
总成绩 = -20.30 + 0.60×(线上平时成绩) + 0.74×(线上章测试)
         + 1.13×(线上期末考) + 0.06×(线下互动) + 0.32×(线下期末考)
         + 0.11×(班级_Sheet7) - 0.58×(班级_Sheet5)
         - 0.14×(班级_Sheet6) - 0.10×(班级_Sheet3)
         - 0.02×(班级_Sheet2) - 0.01×(班级_Sheet4)
```

**分析**：
- `线上期末考`（权重 1.13）和 `线上章测试`（权重 0.74）对总成绩正向影响最大
- `班级_Sheet5` 权重为负（-0.58），说明该班级整体成绩水平相对偏低
- `线下互动` 权重较小（0.06），对总成绩影响有限

### 3.2 模型 B：决策树分类

#### 算法原理

决策树通过特征对数据空间进行递归划分，每个叶节点对应一个预测类别。核心思想是在每个节点选择最优特征进行分裂，使子节点的**不纯度**最小化。本模型使用 **CART 算法**，不纯度衡量采用 **基尼系数（Gini Impurity）**。

$$
\text{Gini}(t) = 1 - \sum_{i=1}^{k} p_i^2
$$

其中 $p_i$ 为节点 $t$ 中第 $i$ 类样本的比例。

#### 等级划分

| 等级 | 分数范围 | 含义 |
|:---:|:--------:|:----|
| A | [90, 100] | 优 |
| B | [80, 89] | 良 |
| C | [70, 79] | 中 |
| D | [0, 69] | 差 |

#### 超参数调优

使用 GridSearchCV（5 折交叉验证）搜索最优参数：

| 参数 | 候选值 | 最佳值 |
|:----|:-----:|:-----:|
| max_depth（最大深度） | 5, 7, 10 | **7** |
| min_samples_split（最小分裂样本数） | 10, 20 | **10** |
| min_samples_leaf（叶节点最小样本数） | 5, 10 | **5** |

> 注：通过设置 `min_samples_leaf=5` 防止过拟合，叶节点至少包含 5 个训练样本，避免学到过于特化的规则。

交叉验证最佳准确率：**0.8544**

#### 训练过程

```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import GridSearchCV

param_grid = {
    'max_depth': [5, 7, 10],
    'min_samples_split': [10, 20],
    'min_samples_leaf': [5, 10],
}
model_b = GridSearchCV(DecisionTreeClassifier(random_state=42, class_weight='balanced'),
                       param_grid, cv=5, scoring='accuracy')
model_b.fit(X_train, y_train_cls)
```

### 4.2 页面一：个体预测

**功能**：教师通过勾选特征、调整滑块或输入数值，系统实时预测其期末总成绩。

**自由勾选特征**：

| 操作 | 说明 |

| 预测模型 | 始终使用 **5 特征完整模型**，未勾选的特征贡献其平均效应 |

这种设计模拟了真实教学场景：教师手头可能只有部分学生信息，信息越完整预测越准确。

**输入组件**：

| 输入项 | 类型 | 范围 |
|:------|:---:|:----:|
| 特征勾选 | 5 个勾选框 | 至少勾选 2 个 |
| 线上平时成绩 | 滑块 + 数字输入框（双向同步） | 0 ~ 50 |
| 线上章测试 | 滑块 + 数字输入框（双向同步） | 0 ~ 10 |
| 线上期末考 | 滑块 + 数字输入框（双向同步） | 0 ~ 40 |
| 线下互动 | 滑块 + 数字输入框（双向同步） | 0 ~ 100 |
| 线下期末考 | 滑块 + 数字输入框（双向同步） | 0 ~ 100 |
| 选择班级 | 下拉框 | 全部 / 1班 ~ 7班 |

> 每个特征同时提供滑块和数字输入框，两者通过 `on_change` 回调双向同步值，解决滑块难以精确定位的问题。

**输出结果**：

| 模型 | 输出 | 展示形式 |
|:----|:----|:--------|

**交互流程**：

```
勾选特征（≥ 2 个）→ 调整滑块/输入数值 → 选择班级（全部/1班~7班）

    → 模型A：predict() → 连续分数 + 当前信息完整度显示

    → 并排展示两个模型的预测结果
```

### 4.4 页面三：模型可视化

**功能**：直观展示模型的内部结构和评估结果。

#### 决策树分支规则图

- 直接渲染 `sklearn.tree.plot_tree` 生成的完整决策树图像
- 每个节点标注：分裂条件、基尼系数、样本数、类别分布
- 颜色越深表示该节点纯度越高（分类越确定）

**读图方法**：
```
根节点（顶部）
  ↓ 判断：线上章测试 ≤ 某值？
  ├── 是 → 左分支 → 继续判断 → 到达叶节点 → 输出等级
  └── 否 → 右分支 → 继续判断 → 到达叶节点 → 输出等级
```
- 从根节点开始，根据每个节点的分裂条件（特征 + 阈值）决定走左分支还是右分支
- 到达叶节点后，该节点的主导类别即为预测结果
- 基尼系数越低（接近 0），该节点纯度越高，分类越确定

#### 模型A评估图

**① 预测值 vs 真实值散点图**

- **横轴**：学生真实总成绩；**纵轴**：模型预测的总成绩
- **红色虚线**：y = x 理想线（预测值 = 真实值）
- **判定标准**：散点越集中在对角线附近，预测越准确。若散点明显偏离对角线，说明模型在该区间存在系统性偏差
- **本模型表现**：R² = 0.9358，散点整体紧贴对角线，说明模型能解释 94% 的总成绩方差

**② 残差分布图**

- **残差** = 真实值 − 预测值（正数 = 模型低估，负数 = 模型高估）
- **理想形态**：残差在 0 附近上下随机波动，无明显喇叭口或弯曲趋势
- **异常信号**：若残差呈现"喇叭口"（预测值越大残差越分散），说明模型存在异方差性；若残差有弯曲趋势，说明线性假设可能不成立
- **本模型表现**：残差均匀分布在 0 两侧，无明显模式，说明线性假设基本成立，模型没有系统性偏差

**③ 回归系数柱状图**

```
线上期末考  ██████████████████████  (1.13)  ← 绿色条 = 正相关
线上章测试  ███████████████    (0.74)
线上平时成绩 ████████████     (0.60)
班级_Sheet5 ████████████  (-0.58) ← 红色条 = 负相关
线下期末考  ██████     (0.32)
班级_Sheet6 ███          (-0.14)
班级_Sheet3 ██           (-0.10)
班级_Sheet7 ██           (0.11)
线下互动    █          (0.06)

```

- **绿色条**：正相关特征——该特征值越大，总成绩预期越高
- **红色条**：负相关特征——该班级整体成绩水平相对偏低
- **条的长度**：权重绝对值越大，该特征对总成绩的影响越显著
- **应用价值**：教师可据此识别哪些指标最能有效拉高学生总成绩（如线上期末考权重 1.13），以及哪些班级需要重点关注（如 Sheet5 负向权重 -0.58）

#### 模型B评估图

**① 混淆矩阵热力图**

- **行**：真实等级；**列**：预测等级
- **对角线数字**：正确预测的样本数（越多越好）
- **非对角线数字**：误判的样本数（越少越好）
- **颜色深浅**：数字越大颜色越深，方便快速定位密集区域

**逐类分析**：

| 等级 | 精确率 | 召回率 | 解读 |
|:---:|:-----:|:-----:|:----|
| A（优） | **0.91** | 0.87 | 没有其他等级被误判为优，但 3 名优生被低估为良 |
| B（良） | 0.88 | 0.88 | 表现最稳定，少数中差交界被误判 |
| C（中） | 0.88 | 0.83 | 召回率偏低，5 名中生被误判为差 |
| D（差） | 0.64 | **0.82** | 差生基本都被识别出，但精确率低说明部分中生被误判为差 |

- **总体**：模型倾向于将边界案例"向上判"（差→中、中→良），使得低等级精确率偏低，但高等级精确率很高

**② 特征重要性图**

- **来源**：基于决策树的 `feature_importances_` 属性，表示每个特征在树分裂中贡献的"信息增益占比"
- **总和为 1**：所有特征重要性之和等于 1，值越大说明该特征在划分数据时越关键
- **本模型表现**：
  - **线下期末考**（0.47）独占将近一半的重要性，是区分成绩等级的最关键指标
  - **线上平时成绩**（0.25）和 **线上章测试**（0.13）也有一定贡献
  - **班级特征**（≈ 0.00）在决策树中几乎未被使用，说明班级差异对等级划分的影响很小

#### 双模型对比总结

在页面底部并排展示两个模型的优缺点：

**模型A（线性回归）**
- 预测精细到具体分数
- 可解释性强（权重系数直观）
- R² = 0.94，误差仅 1.08 分
- 局限：假设特征与目标呈线性关系

**模型B（决策树分类）**
- 等级划分更符合教学管理场景
- 无需线性假设
- 准确率 85.4%（min_samples_leaf=5 防止过拟合）
- 局限：对罕见样本组合可能误判

### 4.5 智能学情预警（拓展功能）

#### 低分预警

当系统预测到学生成绩存在不及格风险时，自动触发红色高亮提示：

| 触发条件 | 预警内容 | 展示形式 |
|:--------|:---------|:---------|
| 回归预测分数 < 60 分 | 🚨 低分预警：预测总成绩仅 XX 分，存在不及格风险！ | 红色边框 + 红色字体高亮卡片 |
| 回归等级为「差」 | 🚨 等级预警：学生成绩处于班级末尾水平 | 同上 |

预警基于 **线性回归（R² = 0.94）** 的预测结果，相比决策树更稳定可靠。

#### 教学干预建议

系统根据学生各特征的实际值，与训练集的 **第 25 百分位（P25）** 对比，自动定位薄弱环节并生成建议：

| 薄弱特征 | 判定条件 | 建议内容 |
|:---------|:---------|:---------|
| 线上平时成绩 | < 40 分 | 建议加强日常作业提交质量和课堂参与度 |
| 线上章测试 | < 10 分 | 建议回顾章节知识点并重新完成测试 |
| 线上期末考 | < 38 分 | 建议重点复习线上课程内容 |
| 线下互动 | < 60 分 | 建议积极出勤并参与课堂讨论 |
| 线下期末考 | < 48 分 | 建议加强线下课程复习和练习 |

建议以 **可展开面板** 的形式展示，每项包含当前值与满分对比，便于教师快速定位问题。

### 4.6 结论

#### 关键发现

1. **线下期末考** 是预测总成绩的最重要特征（两个模型一致认可）
2. **线上平时成绩** 和 **线上期末考** 同样是关键预测因子
3. **渐进预测** 验证了信息越完整预测越准：2特征R²=0.69 → 5特征R²=0.94
4. **班级差异** 对成绩有一定影响，但不如个人成绩重要
5. 线性回归（R² = 0.94）能较准确地预测具体分数，且比决策树更稳定
6. 决策树分类（准确率 85.4%）能有效划分成绩等级
7. **低分预警** 和 **教学干预建议** 功能为教师提供了直观的学情分析工具

#### 模型适用场景

| 场景 | 推荐模型 | 原因 |
|:----|:-------:|:----|
| 需要精确分数预测 | 模型A（线性回归） | MAE 仅 1.08 分 |
| 需要等级评定 | 模型B（决策树分类） | 直接输出优/良/中/差 |
| 需要解释预测依据 | 模型A（线性回归） | 权重系数直观 |
| 处理非线性关系 | 模型B（决策树分类） | 树结构可拟合复杂边界 |

#### 改进方向

1. 引入更多特征（如出勤率、作业提交情况、课堂参与度等）
2. 尝试集成模型（随机森林、XGBoost）以进一步提升准确率
3. 对 D 等级（差生）做针对性优化，提升分类精确率
4. 收集更多跨学期数据，观察成绩变化趋势
